# 수정사항(변경이력) 템플릿

> **사용법**
> 1. 명세서대로 코드를 받은 뒤, 수정/보완이 발생할 때마다 이 파일을 복사해 사용한다.
>    저장 위치: `docs/changes/<원본명세ID>-CL<순번>.md`
>    예: `docs/changes/SPEC-20260422-01-CL01.md`
> 2. **명세서 본문은 가급적 수정하지 않는다.** 명세서를 계속 덮어쓰면 "왜 이렇게 됐는가"의 이력이 사라진다.
>    스펙 자체가 잘못 짜였다면 새 명세서(`SPEC-...-02`)를 발행하고 이 변경 이력에 사유를 적는다.
> 3. 한 변경 = 한 파일이 원칙. 여러 건을 모아 적지 않는다 (사후 추적이 어려워짐).

---

## 0. 메타

| 항목 | 값 |
|---|---|
| 변경 이력 ID | `SPEC-YYYYMMDD-NN-CL<순번>` |
| 원본 명세서 | `SPEC-YYYYMMDD-NN` (링크 또는 경로) |
| 변경 종류 | Bugfix / Spec 보강 / 리팩터 / 정책 변경 / 기타 |
| 발견 경로 | 코드 리뷰 / QA / 운영 장애 / 사용자 피드백 / 본인 검토 |
| 작성자 | |
| 작성일 | |
| 관련 커밋 / PR | (해시 또는 PR 번호) |

## 1. 무엇이 문제였나 (What)

현상을 객관적으로. 추측·해석 빼고 사실만.

예: *"POST /api/recipes 호출 시, 본문에 `authorId` 를 넣어 보내면 그 값이 그대로 저장된다. 명세 §4-2 에 따르면 JWT 의 user_id 만 사용해야 한다."*

---

## 2. 왜 그렇게 됐나 (Why)

근본 원인. "코드가 그랬다" 말고 **왜** 그렇게 짜였는지까지.

예: *"RecipeService 가 RequestDto 를 그대로 Entity 로 매핑하면서 authorId 필드를 덮어쓰지 않았다. 명세서에 \"Body로 받지 않는다\" 라고는 적었지만 DTO 정의에서 명시적으로 제외하지 않아 자동 매핑됨."*

---

## 3. 어떻게 바꿨나 (How)

변경 내용을 코드 단위로.

- **변경 파일**
  - `src/main/java/.../RecipeService.java:42` — `authorId` 를 `SecurityContextHolder` 에서만 읽도록 수정
  - `src/main/java/.../dto/RecipeCreateRequest.java` — `authorId` 필드 제거
- **마이그레이션 / 설정 변경**: 없음
- **테스트**:
  - `RecipeServiceTest#authorIdIgnoredFromBody` 추가

diff 가 짧으면 인라인으로 첨부:
```diff
- recipe.setAuthorId(request.getAuthorId());
+ recipe.setAuthorId(currentUser.getUserId());
```

---

## 4. 명세서에 미치는 영향

- [ ] 원본 명세서 수정 필요 → 새 명세서 발행 (`SPEC-...-02`)
- [x] 명세서 의도와 일치, 구현만 보정 → 명세서 수정 불필요
- [ ] 명세서가 모호했음 → 명세서에 *주석 1줄* 추가 (큰 수정 아님)

명세서를 수정한다면 "어떤 문장을 어떻게 바꿨는지" 변경 전/후를 적는다.

---

## 5. 회귀 테스트 / 검증

이 변경으로 **다른 어디가 깨질 수 있는지** 미리 확인한 결과.

- [ ] 같은 트랜잭션을 쓰는 다른 흐름 검토
- [ ] 동일 DTO 를 쓰는 다른 엔드포인트 검토
- [ ] 통합 테스트 통과
- [ ] 수동 시나리오 검증 (필요하면 어떻게 했는지 기록)

---

## 6. 후속 작업 (Follow-up)

지금 처리하지 않았지만 같이 들여다봐야 할 것.

- [ ] (예) 다른 도메인의 `POST` 엔드포인트들도 동일 패턴인지 점검 필요
- [ ] (예) DTO 관용 매핑 가이드 문서화

---

## 7. 학습 / 회고 (선택)

같은 종류 실수를 반복하지 않기 위한 메모. 짧게 한 줄도 좋다.

예: *"명세에 \"Body로 받지 않는다\" 라고 적을 때는 DTO 필드까지 명시해야 한다. 텍스트로만 적으면 코드 생성 시 누락된다."*
